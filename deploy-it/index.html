Deploy It!

Now that we know what we're going to write, it's time to put it all together into docs for your project!

This tutorial assumes you have a basic knowledge of Git and GitHub, as well as how to write/edit documents in Markdown (it's not hard, don't worry).

Step 1: Set up your docs site

To make our docs today, we're going to be utilizing a template repository on GitHub. This template repository is a version of Just the Docs populated with the templates outlined on the previous page, meaning it'll just take editing a few files to be up and running with a modern and responsive documentation website.

Navigate to the link below and click the green "Use this template" button. This will create a copy of the repository that will serve as the home for your documentation

Get the Template ↓

This repository has GitHub Actions that automatically build the site, meaning that whenever you commit a change to the documentation repository, it automatically rebuilds the site with the new changes. The site is then served to the internet via GitHub Pages for free. To activate this, head to your new repository's settings, go to the Pages section, then make sure that under Build and Deployment, Source is set to Github Actions.

Step 2: Configure the Site

To make the newly copied template site your own, you need to make a few changes to some files.

The first is _config.yml, a file that holds basic configuration data about your site. Make sure to change the title, description, and the url to match the details for your project.

Second of all, there's a Markdown file called `index.md` in the repository. This is the first page that anyone visiting the documentation site will see, so make sure it's nice, friendly, and has links to useful information

Populate the Site

Now that the setup is out of the way, we can get to writing! Utilize the templates provided as part of the template, as well as the resources linked in Dive Deeper, to build a few docs. It may not be the most dense and information heavy documentation site, but remember that having some docs is a lot better than having none.

The README

Next up is the README. This belongs at the root of your project's main GitHub page, and will show up whenever someone visits your project on GitHub. Utilize the template linked below, as well as the resources linked in Dive Deeper, to create a professional-looking README.

Pat Yourself on the Back

If you've made it this far, congrats! You should have a documentation site up and running with a few docs you can build on later, as well as a clean and professional README. You're all set to show the world just how cool your project is!
